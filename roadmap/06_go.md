## Stage 0 — Go Basics

### Learn
- `go mod init`, `go build`, `go run`, `go test`
- Variables, constants, types
- Functions: multiple return values, named returns
- Control flow: `if`, `for`, `switch`
- Pointers: why Go uses them, `&` and `*`
- Packages and imports

### Guided Examples
```go
package main

import (
    "fmt"
    "os"
)

func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, fmt.Errorf("division by zero")
    }
    return a / b, nil
}

func main() {
    result, err := divide(10, 3)
    if err != nil {
        fmt.Fprintf(os.Stderr, "Error: %v\n", err)
        os.Exit(1)
    }
    fmt.Printf("Result: %.2f\n", result)
}
```

### 🟢 Easy Exercises
1. Write a program that swaps two variables using multiple assignment
2. Implement FizzBuzz using a `for` loop and a `switch` statement
3. Write a function that returns both a quotient and remainder
4. Use `const` and `iota` to define an enum-like set of log levels: DEBUG=0, INFO=1, WARN=2, ERROR=3

### 🟡 Medium Exercises
1. Write a variadic function `sum(nums ...int) int` and call it with different numbers of arguments
2. Convert a string to an int safely, handling the returned error properly
3. Write a program demonstrating value vs pointer receivers: a `Counter` struct with `Increment()` method

## Stage 1 — Structs, Methods & Interfaces

### Learn
- Structs: fields, embedding, anonymous fields
- Methods: value vs pointer receivers
- Interfaces: implicit implementation, `interface{}`/`any`
- Type assertion and type switches
- Common interfaces: `io.Reader`, `io.Writer`, `fmt.Stringer`, `error`

### 🟢 Easy Exercises
1. Define a `Server` struct with Name, IP, Status fields and a `String()` method (implement `fmt.Stringer`)
2. Create a `Pinger` interface with `Ping() error` and implement it for `Server`
3. Embed one struct inside another to reuse fields

### 🟡 Medium Exercises
1. Define a `Database` interface with `Connect()`, `Query(string)`, `Close()` and implement for `MySQLDB` and `MongoDB` structs
2. Write a function that accepts an `io.Writer` and works with `os.Stdout`, a `bytes.Buffer`, and a file

### 🔴 Hard Exercises
1. Implement the `sort.Interface` to sort a custom slice of `Server` structs by response time
2. Build a builder pattern for constructing a complex `ClientConfig` struct with functional options

## Stage 2 — Error Handling & Testing

### Learn
- Error handling idiom: `if err != nil`
- Custom error types
- `errors.New`, `fmt.Errorf` with `%w` for wrapping
- `errors.Is`, `errors.As` for unwrapping
- `testing` package: `TestXxx(t *testing.T)`, `t.Error`, `t.Fatal`
- Table-driven tests
- `go test -cover` for coverage
- `testing.B` for benchmarks

### Resources
- https://go.dev/blog/error-handling-and-go
- https://pkg.go.dev/testing

### 🟢 Easy Exercises
1. Write table-driven tests for a `Divide(a, b float64)` function using `testing.T`
2. Create a custom `ValidationError` type that implements the `error` interface with a field name
3. Use `errors.Is` and `errors.As` to check error types in test cases

### 🟡 Medium Exercises
1. Use `fmt.Errorf` with `%w` to wrap an underlying error — unwrap it in the caller with `errors.As`
2. Write a benchmark comparing two string concatenation methods using `testing.B`

### 🔴 Hard Exercises
1. Write an integration test using `net/http/httptest` to test an HTTP handler without a real server
2. Write a fuzz test for a function that parses a custom config format

## Stage 3 — Concurrency

### Learn
- Goroutines: `go func()`
- Channels: buffered vs unbuffered, `<-`, `close()`
- `select` statement
- `sync.WaitGroup`, `sync.Mutex`, `sync.Once`
- `context.Context`: cancellation, timeouts, values
- Common patterns: fan-out/fan-in, worker pool, pipeline

### Resources
- https://go.dev/blog/concurrency-is-not-parallelism
- **Go concurrency patterns (Rob Pike talk):** https://youtu.be/f6kdp27TYZs

### 🟢 Easy Exercises
1. Launch 5 goroutines that each print a number, using `sync.WaitGroup` to wait for all
2. Create a channel, send 3 values from a goroutine, receive them in main
3. Use `select` to receive from whichever of two channels is ready first

### 🟡 Medium Exercises
1. Build a worker pool: N workers pull jobs from a channel and send results to another channel
2. Use `context.WithTimeout` to cancel a long-running goroutine after 2 seconds
3. Use `sync.Mutex` to protect a shared map from concurrent writes — demonstrate the race with `go run -race`

### 🔴 Hard Exercises
1. Build a 3-stage pipeline connected by channels: generator → processor → collector
2. Implement graceful shutdown: catch `SIGTERM`, drain in-flight work, close all goroutines cleanly

- Writing custom controllers

### Resources

### 🟢 Easy Exercises
1. Use `client-go` to list all pods in the default namespace
2. Watch for Pod events and print when a pod is created or deleted
3. Parse a Kubernetes YAML manifest into Go structs

### 🟡 Medium Exercises
1. Write a controller loop that logs when any Pod enters `CrashLoopBackOff`
2. Build a tool that cordons a node and drains its pods gracefully

### 🔴 Hard Exercises
1. Build a minimal Kubernetes Operator with kubebuilder: CRD, controller, and status updates
2. Implement a validating admission webhook that rejects Pods without resource limits
