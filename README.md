# 🚀 Junior DevOps → Platform Engineer Roadmap

> **Your goal:** Stop living paycheck to paycheck. Go from a Junior DevOps role to a Platform Engineer position earning competitive market salaries — both in India and remotely.

---

## 🎯 The Mission

This roadmap is a **complete, self-directed learning system** built for engineers with 0–1 year of DevOps experience who want to reach Platform Engineer level in 18–24 months through deliberate, structured practice.

**Philosophy:** Learn → Understand → Practice → Build → Review — in that exact order, every time.

## 🧠 Learning Philosophy

```
📖 Read the concept
        ↓
🧠 Understand why it exists (not just how)
        ↓
💻 Follow guided examples
        ↓
🟢 Solve Easy exercises (alone, no googling for 30 min first)
        ↓
🟡 Solve Medium exercises
        ↓
🔴 Solve Hard exercises
        ↓
💀 Attempt Expert (benchmark yourself)
        ↓
🏗 Build the DevOps/Platform Project
        ↓
✅ Self-check: can I explain this to someone else?
```

---

## 📅 Recommended Daily Schedule

```
⏰ 30 min — Read & understand the concept
⏰ 20 min — Watch one reference video / read docs
⏰ 40 min — Follow guided examples (type them, don't paste)
⏰ 50 min — Solve exercises (struggle first, then look)
⏰ 30 min — Build something small with today's concept
⏰ 10 min — Review & write one sentence summary
────────────────────────────
Total: ~3 hours/day (consistent beats intense sprints)
```

> **Weekend rule:** Every weekend, build a small project combining the week's topics. No exercises — only building.

---

## 📏 Rules

1. **Never copy-paste code.** Type it. Your hands need muscle memory, not your clipboard.
2. **Struggle first.** Spend 30 minutes minimum on a problem before searching.
3. **Build something every week.** Even if it's a 50-line script. Shipping matters.
4. **Document everything.** Write a short README for every project you build.
5. **Review Fridays.** Every Friday: what did you learn? What's still fuzzy? Plan next week.
6. **No certification before competence.** Get comfortable with the tech, then certify to prove it.
